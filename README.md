# European Art — Object Detection & Style Classification

A two-model computer vision pipeline for analysing European artworks:

1. **YOLOv8** — detects iconographic objects inside paintings (134 classes: 80 COCO + 54 DEArt-exclusive).
2. **MobileNetV3** — classifies the art style of a painting across 27 WikiArt categories.

Both models are exported to ONNX with optional INT8 quantisation for lightweight CPU/edge deployment.

---

## Datasets

| Dataset | Task | Source |
|---|---|---|
| [biglam/european_art](https://huggingface.co/datasets/biglam/european_art) | Object detection | HuggingFace Hub |
| [huggan/wikiart](https://huggingface.co/datasets/huggan/wikiart) | Style classification | HuggingFace Hub |

Both datasets are downloaded and cached automatically by HuggingFace `datasets` on first run.

## Setup

```bash
# 1. Create a virtual environment (recommended)
python -m venv .venv && source .venv/bin/activate   # Linux / macOS
# python -m venv .venv && .venv\Scripts\activate    # Windows

# 2. Install dependencies
pip install -r requirements.txt
```

> **GPU note**: the `requirements.txt` installs the CPU-only PyTorch build by default.
> For GPU training, install the CUDA wheel **before** running pip:
> ```bash
> # Example for CUDA 12.1 — pick your version at https://pytorch.org/get-started/locally/
> pip install torch torchvision --index-url https://download.pytorch.org/whl/cu121
> pip install -r requirements.txt
> ```
> All scripts fall back to CPU automatically when CUDA is unavailable.

## Workflow

### 1 — Prepare the dataset

`data_process.py` is organised into four independent stages.
Run them all in sequence for a fresh setup:

```bash
python data_process.py
```

Or run a single stage in isolation:

```bash
python data_process.py --stage build      # convert DEArt → YOLO format + dataset.yaml
python data_process.py --stage augment    # merge COCO val2017 into the dataset
```

| Stage | What it does | Key outputs |
|---|---|---|
| `build` | Converts bounding boxes from COCO JSON to normalised YOLO `.txt` files, saves images, and generates `dataset.yaml` | `dataset_yolo/` |
| `augment` | Downloads 5,000 COCO val2017 images and merges them into `dataset_yolo/` for better generalisation on shared classes | Updated `dataset_yolo/` |

Additional options:

```bash
# Use a custom output directory and validation split
python data_process.py --stage build --base-dir my_dataset --val-split 0.15
```

---

### 2 — Train the object detector

```bash
python train_yolo.py
```

Runs two sequential training stages:

| Stage | Epochs | Frozen layers | LR | Purpose |
|---|---|---|---|---|
| 1 — Head-only | 20 | 0–21 (backbone) | 1e-2 | Warm up the detection head without disturbing COCO pre-trained features |
| 2 — Full fine-tune | 50 (+ early stop) | none | 1e-4 | Adapt the whole network to the art domain |

Best weights are saved to `runs/european_art/stage2_full/weights/best.pt`.

---

### 3 — Train the style classifier

```bash
python train_class.py
```

Trains a MobileNetV3-Small (27-class) on WikiArt for 40 epochs with:
- **RandAugment** + random crop / flip for data augmentation.
- **AdamW** + **Cosine Annealing** LR schedule.
- **Label smoothing** (0.1) for better calibration.

| Output | Description |
|---|---|
| `models/wikiart_model_aug_only.pth` | Best checkpoint (saved whenever val accuracy improves) |
| `results/training_metrics.csv` | Per-epoch loss and accuracy |
| `results/confusion_matrix_seaborn.png` | Normalised confusion matrix |
| `results/class_training_curves_clean.png` | Smoothed loss and accuracy curves |

---

### 4 — Export to ONNX

**Style classifier:**

```bash
python export_class_onnx.py
# outputs: style_classifier.onnx   style_classifier_int8.onnx
```

**Object detector:**

```bash
# On the first run, uncomment the Step 1 block inside export_yolo_onnx.py, then:
python export_yolo_onnx.py
# outputs: best.onnx   best_int8.onnx
```

INT8 models are ~4× smaller than FP32 with minimal accuracy loss, making them well-suited for CPU or edge deployment.
